Data & API

Every score, description, and source on this site is available for research use — as plain static files, and as a public read-only REST API with CSV export, queryable score history, an accumulating sources database, and the verified policy-initiative records that ground the scores. See the methodology for how scores are constructed.

Static files

The simplest way to get the full dataset — one HTTP GET, no key, CDN-served. These files are the published snapshot of the database and update together, monthly.

FileContents
/scores.csv 196 countries × 6 score columns (1–5, quarter-point decimals), last-updated date, data version. Columns: Country, Regulation Status, Policy Lever, Governance Type, Actor Involvement, Average Score, Enforcement Level, Last Updated, Data Version.
/regulation_data.csv Per-country prose for the five dimensions, key legislation, pipe-separated source URLs, confidence (high/medium/low), last-updated.
/history.json Change-point score snapshots per country (a snapshot is appended only when a score changes), powering the timeline.
/data/subscores.json Methodology-v2 audit trail: the four named sub-indicators (integers 1–5) behind each dimension score.
/data/country_iso.json ISO 3166-1 alpha-2/alpha-3/numeric codes for every dataset country name.

The full dataset, one call

curl "$BASE/public_export?select=*" \
  -H "apikey: $KEY" -H "Authorization: Bearer $KEY"

Add Accept: text/csv for a CSV download instead of JSON:

curl "$BASE/public_export?select=country,iso3,avg_score,confidence" \
  -H "apikey: $KEY" -H "Authorization: Bearer $KEY" \
  -H "Accept: text/csv" -o ai-regulation.csv

Queryable tables

EndpointContents
/public_exportView joining countries × scores × summaries — the whole dataset flat.
/countriesCanonical names, ISO codes, and name aliases.
/country_scoresCurrent scores incl. the sub-indicator breakdown as JSON (subscores).
/country_summariesCurrent per-dimension prose, key legislation, raw source list.
/score_historyChange-point snapshots — every score a country has ever carried, with dates.
/sources · /sources_exportThe accumulating sources database: every cited URL, classified (official / intergovernmental / academic / …), with first/last-seen dates; the export view adds which countries cite it.
/policy_initiativesVerified policy-initiative records (OECD.AI Policy Navigator) that ground the scoring — name, type, binding status, years, overview, source link.
/research_runsProvenance: every research run with model, prompt version, whether it was evidence-grounded, and counts.

Example queries

PostgREST filter syntax works on every endpoint:

# Germany's full score history
curl "$BASE/score_history?select=snapshot_date,scores,countries!inner(name)&countries.name=eq.Germany&order=snapshot_date" \
  -H "apikey: $KEY" -H "Authorization: Bearer $KEY"

# Binding initiatives since 2024, most recent first
curl "$BASE/policy_initiatives?select=name,start_year,binding,source_url,countries(name)&binding=eq.Binding&start_year=gte.2024&order=start_year.desc" \
  -H "apikey: $KEY" -H "Authorization: Bearer $KEY"

# Countries citing a given source
curl "$BASE/sources_export?url=eq.https://eur-lex.europa.eu/eli/reg/2024/1689/oj" \
  -H "apikey: $KEY" -H "Authorization: Bearer $KEY"
Practical notes: responses are capped at 1,000 rows per request — page with Range headers or limit/offset. The API is best-effort infrastructure for research convenience; for archival reproducibility, cite the static files (versioned in the git repository) or keep a copy of your extract.

Attribution & citation

Policy-initiative records include data from the OECD.AI Policy Observatory (Policy Navigator / GAIIN), used with attribution per OECD terms: OECD.AI Policy Observatory, oecd.ai, accessed on the date shown per record.

To cite the map itself, use the Share button in the app for a formatted APA/Chicago/MLA citation of the exact view you're referencing, or the per-country Cite button. Scores are model-assigned (see the methodology and per-run provenance in /research_runs) — cite them as estimates, and verify claims against the primary sources each entry links.
